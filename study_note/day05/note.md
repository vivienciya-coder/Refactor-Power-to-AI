## 一、关键词
-  In-Context Learning  API	不重新构建 (No Training)，在调用接口时动态注入逻辑。
-  Few shot Learning 给模型 3-5 个“样板代码”，让它通过模式识别完成任务。
###解决了什么问题
- 传统的机器学习需要训练大量的数据才可以达到推理效果 而少样本提示和上下文学习通过几个例子就可以使模型更好的推理
## 核心技术
-  OpenAI GPT Models 它是最终执行代码、理解语义的地方。没有它，上层建筑跑不动。
-  LangChain 它不产生能力，它只是能力的搬运工。它把底层的 LLM 调用、缓存、流式输出、向量数据库等功能全封装好了。
-  PromptTemplate  目的是为了“重用”逻辑，不需要每次都手写一长串 Prompt。
-  LLMChain  它负责把“组件 (Prompt)”和“执行引擎 (LLM)”连接起来。类似于前端的一个 handleSubmit 函数，它知道从哪拿数据（Prompt），发给谁（LLM），然后返回结果。
## 具体实现
1. 少量样本学习
   1. 用少样本去执行一个情感分析任务
   2. 这样的少样本结构化提示应该怎么写
   3. 理解大模型是如何通过少样本的例子来归纳从统一性 从而得到输出的
